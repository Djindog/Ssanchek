# Ssanchek 최적화 알고리즘 설계 문서

## 프로젝트 개요

알라딘 중고책 가격 비교 서비스. 사용자가 위시리스트에 책을 추가하면, 예산 범위 내에서 할인율이 높은 구매 조합을 찾아주는 서비스.

---

## 문제 정의

### Input
- 위시리스트: 책 N권 (N ≤ 40)
- 각 책마다 구매 옵션 ~50개:
  - 새책 1개 (무료배송, 정가 대비 10% 할인 고정)
  - 알라딘 중고 1개
  - 광활한우주점 ~10개 지점
  - 판매자 중고 ~40개
- 예산 범위: [min, max] (예: 5만원 ~ 10만원)
- 제약조건:
  - 최소 책 상태 등급 (상/중/하)
  - 최대 배송기한 (일)
  - 필수 포함 책 목록 (무조건 사야 하는 책들)

### Output
- 예산 범위 내 조합들 중 할인율 높은 순 Top K개
- 각 조합 = 어떤 책들을 어느 판매처에서 살지 + 총 비용 + 할인율

### 할인율 계산
```
할인율 = (Σ정가 - Σ실구매가) / Σ정가
실구매가 = Σ책가격 + Σ판매처별배송비 (응집도 반영)
정가 = 알라딘 API 제공 정가
```

---

## 어휘 정의

| 용어 | 정의 |
|------|------|
| 판매처 | 광활한우주점 지점 + 판매자 중고 (새책 제외 - 배송비 무료라 추적 불필요) |
| 멀티판매처 | 위시리스트에서 2권 이상 보유한 판매처 |
| 싱글판매처 | 위시리스트에서 1권만 보유한 판매처 |
| 공유판매처 | 어떤 책에 대해 다른 책도 함께 보유한 판매처 (= 멀티판매처) |

---

## 배송비 구조

| 판매처 종류 | 배송비 |
|------------|--------|
| 새책 | 무료 |
| 알라딘 중고 | 고정 (확인 필요) |
| 광활한우주점 | 지점당 2500원, 해당 지점에서 20000원 이상 구매 시 무료 |
| 판매자 중고 | 판매자별 고정 (보통 2500~3000원) |

**핵심:** 같은 판매처에서 여러 책을 사면 배송비 1회만 청구.
따라서 판매처 응집도(같은 판매처에서 많이 살수록)가 높을수록 배송비 절감.

---

## 전처리 단계

알고리즘 시작 전 다음을 수행:

### 1. 옵션 필터링
- 최소 상태 등급 미달 옵션 제거
- 최대 배송기한 초과 옵션 제거
- 새책 가격(배송비 포함, 10% 할인)보다 비싼 중고 옵션 제거
  - 기준가 = 새책 정가 × 0.9 (무료배송이므로 그대로)

### 2. 멀티판매처 식별 (Union-Find)
```
for 각 판매처:
  그 판매처가 보유한 책 목록 확인
  2권 이상이면 → 멀티판매처로 분류
  1권이면 → 싱글판매처로 분류
```

### 3. 각 책에 대한 옵션 압축
각 책마다:
- 멀티판매처 옵션들: 각각 개별 유지 (배송비 영향 있음)
- 싱글판매처 옵션들: 최저가 1개로 압축 (다른 책 배송비에 영향 없음)
- 새책: 별도 옵션 1개
- 안 사기: 0원 옵션 1개

압축 후 각 책의 옵션 수 = 멀티판매처 수(M_i) + 2

## 핵심 알고리즘: Sparse DP

### 상태 정의
```
dp[(예산, 멀티판매처 사용 집합)] = 최대 할인율
```

- 예산: 100원 단위로 나눔 (예산 40만원 → 4000단계)
- 멀티판매처 사용 집합: 지금까지 사용한 멀티판매처들의 집합
- Sparse: 실제로 도달한 상태만 딕셔너리에 저장

### 이론적 최대 상태 수
```
4000(예산) × 2^M (M = 멀티판매처 총 수)

M = 10: 4000 × 1,024 = 4,096,000 → 빠름
M = 15: 4000 × 32,768 = 131,072,000 → 가능
M = 20: 4000 × 1,048,576 = 42억 → 위험 (Sparse로 완화 가능)
M > 20: 불확실, 데이터 필요
```

### DP 진행 방식
```
초기 상태:
dp = { (0원, frozenset()): 0% }

책을 1권씩 순서대로 처리:
  for 현재 모든 상태 s in dp:
    for 이 책의 각 옵션 o:
      
      케이스 1 - 이 책에 멀티판매처 없음:
        옵션 2개만 시도:
          - 안 사기: 상태 그대로
          - 싱글/새책 최저가: 예산만 차감
        (멀티판매처 집합 변화 없음 → 상태 수 최대 2배)

      케이스 2 - 이 책에 멀티판매처 있음:
        옵션 (M_i + 2)개 시도:
          - 안 사기
          - 싱글/새책 최저가
          - 멀티판매처 각각: 예산 차감 + 집합에 추가
        
      새 상태 key = (새 예산, 새 멀티판매처 집합)
      기존에 같은 key가 있으면 → 할인율 높은 것만 유지
      없으면 → 새로 추가
```

### 배송비 계산
```
총 비용 = Σ책가격 + Σ판매처별 배송비

배송비 계산:
  for 사용된 각 멀티판매처 p:
    if p가 우주점:
      해당 지점 구매액 합계 >= 20000 → 배송비 0
      else → 배송비 2500
    else (판매자 중고):
      배송비 = p의 고정 배송비

  + 싱글판매처 배송비 (선택된 경우)
  + 알라딘 중고 배송비 (선택된 경우)
  + 새책 배송비 = 0
```

### 필수 포함 책 처리
```
1. 필수책들의 최적 옵션 먼저 확정
2. 필수책만으로 이미 예산 초과 → 불가능 반환
3. 남은 예산 = [min - 필수책비용, max - 필수책비용]
4. 선택책들은 남은 예산 내에서 탐색
```

### 예산 범위 처리
```
DP 완료 후:
  모든 상태 중 예산 범위 [min, max] 내인 것만 필터링
  할인율 높은 순으로 Top K개 반환
```

## 검토한 다른 풀이 방식들

### 1. 순수 0/1 Knapsack
- 배송비 의존성(판매처 응집도) 반영 불가
- 탈락

### 2. Multiple Choice Knapsack
- 각 책마다 옵션 그룹에서 1개 선택하는 구조로 우리 문제와 유사
- 배송비 의존성 반영 불가
- 탈락 (단, 초기 해 생성에는 활용 가능)

### 3. 상태 공간 전체 판매자 집합 추적
- 판매자 수천명 → 2^수천 상태 → 완전 불가능
- 탈락

### 4. 시뮬레이티드 어닐링 (SA)
- 초기 해에서 출발, 랜덤하게 책 추가/제거/판매처 교체
- 배송비 정확히 계산 가능
- 멀티판매처 수에 관계없이 동작
- 정확한 최적해 보장 안 됨
- **Sparse DP가 실패할 경우 (M > 20) 폴백으로 사용**

### 5. 라그랑지안 완화
- 배송비를 목적함수에 패널티로 흡수
- 구현 복잡, 수렴 보장 안 됨
- 보류

## 알라딘 API 정보

### 사용 가능한 API
- 상품 검색: `ItemSearch.aspx` (제목/저자/ISBN 검색)
- 상품 조회: `ItemLookUp.aspx` (ISBN으로 상세 조회)
- OptResult=usedList 파라미터로 중고 정보 포함 가능

### API 응답에서 제공되는 중고 정보
```
usedList.aladinUsed.minPrice  → 알라딘 직영 중고 최저가
usedList.aladinUsed.link      → 알라딘 직영 중고 리스트 URL
usedList.userUsed.minPrice    → 판매자 중고 최저가
usedList.userUsed.link        → 판매자 중고 리스트 URL
usedList.spaceUsed.minPrice   → 우주점 최저가
usedList.spaceUsed.link       → 우주점 리스트 URL
```

### API 한계 → 크롤링 필요
API는 판매처별 최저가와 링크만 제공. 개별 판매자 정보, 배송비, 책 상태 등급은 해당 링크 크롤링으로 수집.

### 크롤링 수집 항목
- 판매자명 (멀티판매처 식별용)
- 가격
- 책 상태 등급 (상/중/하)
- 배송비
- 배송기한
- 판매자 링크
- 우주점 지점명 (지점별 배송비 계산용)
